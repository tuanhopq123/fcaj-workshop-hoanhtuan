## Overview

This section summarizes my 12-week internship progress throughout the AWS Cloud Internship Program. During the internship, I gradually developed a complete cloud-native solution named **Smart Home Energy Waste Monitoring & Alert System using Virtual Sensors on AWS**.

The project started with fundamental AWS services and progressively evolved into a production-oriented cloud architecture following AWS Well-Architected best practices. Alongside the project, I completed numerous AWS hands-on laboratories covering networking, security, serverless computing, storage, monitoring, databases, analytics, and cost optimization.

Each week focused on a specific objective, allowing me to continuously improve both practical AWS skills and software engineering knowledge.

---

## Internship Timeline

### Week 1
- Introduction to AWS Cloud and AWS Academy environment.
- Learned AWS global infrastructure and IAM fundamentals.
- Created the initial AWS environment for the project.

### Week 2
- Designed the initial architecture of the Smart Home Energy Waste Monitoring & Alert System.
- Studied Amazon DynamoDB and basic serverless concepts.
- Completed introductory AWS labs.

### Week 3
- Built the first Serverless backend using AWS Lambda and Amazon DynamoDB.
- Designed database schema and API interaction flow.
- Continued hands-on AWS practices.

### Week 4
- Developed business logic using AWS Lambda.
- Integrated Lambda with DynamoDB.
- Practiced Hybrid DNS, VPC Peering, and AWS Transit Gateway.

### Week 5
- Configured Amazon API Gateway.
- Created Amazon S3 buckets for static resources.
- Completed AWS Backup, SNS, and Amazon S3 labs.
- Participated in AWS Community Day to learn Cloud Computing and Generative AI.

### Week 6
- Integrated Amazon CloudWatch for centralized logging and monitoring.
- Performed end-to-end API testing using Postman.
- Practiced AWS Security Hub, Lambda cost optimization, and Resource Tagging.

### Week 7
- Started writing the Workshop documentation.
- Designed the Workshop website interface.
- Prepared AWS resource cleanup scripts.
- Learned IAM Resource Tags and Permission Boundaries.

### Week 8
- Completed the Workshop documentation.
- Added API testing validation.
- Integrated AWS KMS and CloudTrail concepts into the system design.
- Learned Amazon RDS, KMS, CloudTrail, Athena, and IAM Roles.

### Week 9
- Completed project self-evaluation and internship feedback.
- Designed a future Data Pipeline architecture using AWS Glue, Athena, Amazon S3, and Amazon QuickSight.
- Practiced AWS Data Lake and Business Intelligence solutions.

### Week 10
- Redesigned the entire system into a production-ready 3-Tier Architecture.
- Added Amazon RDS, Amazon DocumentDB, Amazon ElastiCache, CloudFront, Secrets Manager, Cloudinary, and Casso integration.
- Studied AWS Well-Architected Framework to improve system scalability and security.

### Week 11
- Developed the automated Report Generator module using AWS Lambda.
- Configured secure IAM policies following the Least Privilege principle.
- Implemented environment variables and automated report storage on Amazon S3.

### Week 12
- Performed comprehensive end-to-end validation of the entire project.
- Finalized Workshop documentation, presentation slides, deployment scripts, and cleanup scripts.
- Reviewed all AWS knowledge acquired during the internship.
- Successfully completed and submitted the final project.
